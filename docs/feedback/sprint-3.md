## 検証メモ

- 上訴チェーンは次の 3 件を仕込んでテストに使った:
  - `sample-002`（東京地裁・所得税） → `sample-005`（東京高裁） → `sample-007`（最高裁）
  - `sample-010`（法人税・地裁） → `sample-012`（法人税・高裁）
- `sample-005` を開くと「下級審 → sample-002」「上訴先 → sample-007」の双方向リンクが両方表示される
- タブは URL クエリ `?tab=` で状態化されているので、サーバーコンポーネントのまま動作（JSクライアント不要）

## 良かった点

- タブを URL クエリで駆動しているため、戻る・進む・直リンク・共有がすべて自然に動く
- 関連事案リンクが「下級審 →」「上訴先 →」というラベル付きで方向性が直感的
- `next/link` で遷移がSPA的に速い
- 404 ページも一覧に戻る導線を持っていて迷子にならない
- data-testid を要素レベルで揃えていて、自動テストの回帰検証がやりやすい

## 改善要望（必須ではない / 次スプリント以降）

- 控訴・上告チェーンが 3 段以上ある場合、「最下級審 ↓ … ↓ 最終審」のパンくず的表示があると便利
- 全文タブはまだプレーンテキストなので、将来的に見出し・引用・強調が欲しい（Markdown レンダリングなど）
- タブ切り替え時にページ全体のスクロールがリセットされないように `scroll={false}` にしてある（確認済）
