# 税務判例サーチ（仮称） - 仕様書

> このファイルは Planner のみが書き込みます。
> 基盤メモ: AI秘書/3_resources/business-ideas/事業案-判例検索ツール.txt

## 概要

税務で困っている一般人（個人事業主、法人経理担当、税務調査を受けた人、相続で困っている人、税理士試験受験生など）が、自分の状況に近い税務裁判例・裁決例を「キーワード」または「自然な質問文」で探せる、検索専用の無料Webサービス。

既存の税理士向け有料DB（TAINS、TKCローライブラリー等）は会費制・月額制で一般人は手が届かない。一方、国税不服審判所や裁判所の無料公式DBはUIが古く、初学者には使いこなせない。本サービスはこの空白地帯を「公開情報を見やすく整理するだけ」の立ち位置で埋める。

**重要な設計原則: アドバイスは一切出さない。検索・表示のみ。** これにより弁護士法72条（非弁行為）リスクを回避し、図書館・Wikipedia と同じ情報提供サービスとして成立させる。

## ターゲットユーザー

- **メイン:** 一般人（税務で困っている個人事業主・法人経理担当・税務調査を受けた人・相続関係者）
- **サブ:** 税理士志望者・税理士試験受験生・若手税理士

## コア機能一覧

| # | 機能名 | 説明 | 優先度 |
|---|--------|------|--------|
| 1 | ジャンルフィルター | 税務のサブカテゴリ（所得税/法人税/相続税/消費税/地方税/その他）で絞り込み | Must |
| 2 | キーワード検索 | 事件概要・要旨に対するフルテキスト検索 | Must |
| 3 | 一覧表示 | 事件番号・日付・裁判所/審判所・ジャンル・最終結果（原告勝訴/一部/敗訴）を一覧化 | Must |
| 4 | 詳細ページ | 要旨／全文／関連法令を切り替え表示。控訴・上告の結果を紐付けて表示 | Must |
| 5 | 自然言語検索 | 「困ってること」を自然文で入力すると関連裁判例がマッチされる（差別化の核） | Must |
| 6 | 免責表示 | 全ページに「情報提供のみ。法的判断は専門家に相談を」と明記 | Must |

## 非機能要件

- **非弁行為リスク回避:** サービスはあくまで公開情報の検索・表示のみ。AIは「検索マッチング」にのみ使用し、「あなたは勝てる」等の解釈・助言は絶対に出さない。
- **無料:** MVPの全機能を無料提供。将来的な広告表示は別途検討。
- **モバイル対応:** スマホでサッと調べられる体験（既存サービスの空白地帯）。
- **免責:** 全ページに「本サービスは情報提供を目的とし、法的助言ではありません。具体的な判断は税理士・弁護士にご相談ください。」と常時表示。

## データ

- **初期投入件数:** 50件程度（検証目的なので小さく始める）
- **ソース:**
  - 国税不服審判所 公表裁決事例 https://www.kfs.go.jp/service/
  - 裁判所 裁判例情報（税務関連のみ） https://www.courts.go.jp/app/hanrei_jp/search1
- **サブカテゴリ:** 所得税 / 法人税 / 相続税 / 消費税 / 地方税 / その他
- **1件あたりの保持項目:**
  - 事件番号 / 裁判所または審判所 / 判決日・裁決日
  - サブカテゴリ
  - 事件名・争点
  - 要旨（数百字）
  - 全文 または 全文URL
  - 最終結果（原告勝訴 / 一部認容 / 原告敗訴 / 請求棄却 等）
  - 関連法令
  - 控訴・上告の有無と結果（紐付け）

## スプリント計画

### Sprint 1: データ取得基盤とDB

**ゴール:** 50件の裁決・判例データを構造化して保持し、全件取得できる最小APIを用意する。

**機能:**
- [ ] DBスキーマ設計（cases テーブル＋subcategories / results 等の列挙）
- [ ] 国税不服審判所・裁判所サイトからのデータ取得スクリプト（Python もしくは TS）
- [ ] 50件程度（6サブカテゴリにできるだけ偏りなく）を手動確認しつつインポート
- [ ] GET /api/cases（全件 or 件数制限付き）で一覧JSONを返す
- [ ] トップページで、データ件数が確認できる最小表示

**受け入れ基準:**
- DBに50件前後のレコードが入っており、各レコードに「事件番号・日付・裁判所/審判所・サブカテゴリ・要旨・最終結果」が必須項目として埋まっている
- API `/api/cases` が200を返し、50件前後のJSON配列を返す
- アプリがエラーなく起動し、トップページで件数（例: 「収録件数: 50件」）が確認できる
- 全ページのフッターに免責文言「本サービスは情報提供を目的とし、法的助言ではありません。」が表示されている

---

### Sprint 2: ジャンルフィルター + キーワード検索 + 一覧表示

**ゴール:** 一般人が「カテゴリ絞り込み」と「キーワード」で裁判例を探し、結果を一覧で読める状態にする。

**機能:**
- [ ] 6サブカテゴリのフィルターUI（複数選択可 or 単一選択）
- [ ] キーワード検索入力欄（要旨・事件名・争点に対するフルテキスト検索）
- [ ] 検索結果一覧（事件番号 / 日付 / 裁判所・審判所 / サブカテゴリ / 最終結果バッジ）
- [ ] 最終結果バッジは色分け（例: 原告勝訴=緑、一部認容=黄、原告敗訴=グレー）
- [ ] 検索結果0件時の空状態表示
- [ ] モバイル幅でレイアウトが崩れない

**受け入れ基準:**
- サブカテゴリ「相続税」を選択すると相続税の事例のみが表示される
- キーワード「売上除外」で検索すると要旨に該当語を含むレコードだけが返る
- 一覧の各行に「事件番号・日付・裁判所/審判所・サブカテゴリ・最終結果」が見えている
- 検索条件に一致するレコードが0件のとき、「該当する裁判例が見つかりませんでした」と表示される
- スマホ幅（375px）でレイアウトが横スクロールなしに閲覧できる
- 免責文言が引き続き全ページで表示されている

---

### Sprint 3: 詳細ページ（要旨 / 全文 / 関連法令）

**ゴール:** 一覧からクリックで詳細に遷移し、要旨・全文・関連法令を切り替えて読める状態にする。

**機能:**
- [ ] 一覧行クリックで詳細ページへ遷移
- [ ] 詳細ページに「要旨 / 全文 / 関連法令」のタブ
- [ ] 最終結果・控訴/上告の有無を上部に表示
- [ ] 控訴・上告の紐付け（同一事案の下級審・上級審を互いにリンク）
- [ ] 一覧に戻るナビゲーション

**受け入れ基準:**
- 一覧から任意の事例を開くと、その事例の詳細ページが表示される
- 要旨・全文・関連法令がタブで切り替えられ、それぞれ内容が表示される
- 控訴・上告がある事例では、関連事案へのリンクが表示され、クリックで遷移できる
- 詳細ページ上部に最終結果（例:「最終: 東京高裁 原告勝訴」）が明示されている
- 詳細ページにも免責文言が表示されている

---

### Sprint 4: 「困ってることから探す」自然言語検索

**ゴール:** 自然文の質問から関連裁判例をマッチングして表示する（差別化の核）。ただし解釈・助言は絶対に出さない。

**機能:**
- [ ] トップページに「困ってることを書いてみる」自然文入力欄
- [ ] 入力をClaude API（embeddings もしくは検索特化プロンプト）で裁判例にマッチング
- [ ] 上位5〜10件を「関連しそうな裁判例」として一覧表示
- [ ] 結果一覧には「これは情報提供です。あなたの事案に当てはまるかは専門家にご相談ください」と明示
- [ ] 入力に対しAIが「勝てる/負ける」「あなたは〜すべき」等の解釈を返していないこと

**受け入れ基準:**
- 「フリーランスで経費にした車の購入費を否認された」と入力すると、所得税・法人税系の関連裁判例が返る
- 「相続で土地の評価を巡って争った事例」と入力すると、相続税系の関連裁判例が返る
- AIが返すのは「裁判例リスト」のみであり、アドバイス文・結論・勝敗予想を一切含まないこと（プロンプト上禁止、出力をテストで検証）
- 結果ページに「情報提供のみ。法的判断は専門家に相談を」が目立つ位置に表示されている
- レスポンスが20秒以内に返る
- 回帰: Sprint 1〜3 の機能（カテゴリフィルター・キーワード検索・一覧・詳細）がすべて引き続き動作する

## やらないこと（MVP後に回す）

- ユーザー登録・ログイン
- お気に入り・通知・アラート
- AIによる要約・解釈・アドバイス・勝敗予想
- 弁護士紹介・マッチング
- 広告表示（ユーザー獲得後に再検討）
- 他ジャンル（相続、離婚、労働、刑事等）への拡張
- 全文PDF保存・印刷最適化
- B2B向けAPI提供

## 検証ゴール

- 自然言語で聞ける検索があれば、一般人でも税務裁判例を探す気になるか
- 「検索だけ・助言なし」でも情報としての価値を感じるか
- 欠けている機能・カテゴリは何か
